1.1 What is SNAP?

Back to the beginning ⇡

What is SNAP?

The Sentinel Application Platform (SNAP) is a free, open-source software developed by Brockmann Consult, SkyWatch, Sensar, and C-S, and supported by the European Space Agency (ESA) specifically for processing and analyzing data from the Sentinel satellite missions, as well as other Earth observation datasets. It is designed to support data from Sentinel-1 (radar), Sentinel-2 (optical), and Sentinel-3 (optical, altimetry, and radiometry) satellites, part of the Copernicus program, which monitors Earth's environment.

  • SNAP offers several toolboxes, designed to work with specific data types and tasks: Optical Toolbox: processes optical satellite data, such as images from Sentinel-2 and Landsat

  • Microwave Toolbox: processes microwave data, including Sentinel-1 radar data. Enabling tasks like synthetic aperture radar (SAR) processing, speckle filtering, and radar data interpretation

  • SMOS Toolbox: processes data from the Soil Moisture and Ocean Salinity (SMOS) satellite.

Beyond Sentinel, SNAP supports data from other Earth observation missions, such as Landsat, MODIS, PROBA-V and Envisat.

SNAP includes a wide range of tools for remote sensing image processing such as radiometric and geometric correction, atmospheric correction, spectral indices computation, filtering and image enhancement. It provides tools for visualizing satellite images, working with RGB composites, analyzing histograms, and overlaying different bands to interpret land, water, and vegetation features.

SNAP's architecture allows users to extend functionality with custom plugins and scripts (Python, Java), allowing for more specialized analysis and research applications. SNAP offers options for automating tasks, allowing users to process large volumes of satellite data with batch processing and command-line tools. This is especially useful for time-series analysis and monitoring environmental changes.

SNAP can be used in a variety of application areas, including but not limited to:

  1. Land Monitoring: Analyses of land cover and land use changes, vegetation health, and soil properties. This includes agricultural monitoring, deforestation tracking, and urban development analysis.

  2. Oceanography: Monitoring of sea surface temperature and marine pollution, which is crucial for understanding marine ecosystems

  3. Atmospheric Studies: Tracking atmospheric conditions, air quality, and greenhouse gas concentrations, which helps in climate change research and pollution control.

  4. Disaster Management: Assessing and responding to natural disasters like floods, wildfires, and earthquakes.

  5. Hydrology: Monitoring water bodies, including river discharge, water quality, and glacier dynamics, which is important for water resource management.

  6. Biodiversity and Conservation: Mapping habitats and tracking changes in biodiversity, which is useful for conservation efforts and habitat restoration projects.

  7. Urban Planning: Analyzing urban sprawl, infrastructure development, and environmental impacts of urbanization.

  8. Climate Research: Studying the impacts of climate change on various ecosystems and human activities, including long-term environmental monitoring and climate modeling.

1.2 SNAP Interface

Back to the beginning ⇡

The Sentinel Application Platform has a user-friendly interface designed to facilitate the visualization, analysis, and processing of Earth observation data.

The Sentinel Toolbox Menu Bar contains: File Menu, Edit Menu, View Menu, Analysis Menu, Layer Menu, Vector Menu, Raster Menu, Optical Menu, Radar Menu, Tools Menu, Window Menu, Help Menu.

SNAP main interface

The Image View on the right side of program window displays the sample values of raster datasets such as bands and tie-point grids as an image. The view uses tiled images and image pyramids, and therefore has a good image display performance and memory usage. Additionally the view is enhanced with a navigation control which can be used to zoom and rotate the image.

A new Image View is simply created by double-clicking on a tie-point grid or spectral/geophysical band. You can open as many images as your computer's RAM allows. After you have opened an Image View you can inspect the images with the Navigation Window. After you have opened an Image View you can modify the colors of the image using the Colour Manipulation window or overlay an opaque or semi-transparent data mask with the Mask Manager. Both tool windows operate in non-modal mode, which means they float over the main frame and you can place them somewhere on your desktop.

Image View Window

If you create a new, open or import an existing data product, the Sentinel Toolbox will put it into it's open product list in the Product Explorer window on the left side of the program window. The product list is a tree view with up to four root nodes for each open product - Bands, Vector data, Tie-point grids and other data.

Product Explorer Window

If you click on the tab 'Pixel View', pixel information will be displayed while you move the mouse over the band image view.

Pixel Info View

The Navigation Window located under the Product Explorer tab is used to move the viewport of an Image View, to zoom in and out of it and to rotate the image in steps of 5 degrees using the spinner control below the image preview. The current viewport is depicted by a semi-transparent rectangle which can be dragged in order to move the viewport to another location. It also provides a slider used to zoom in and out of the view.

Navigation Window

The Tool Windows of the Sentinel Toolbox provide access to tools, as the name says. These windows can be floating, docked or tabbed. This docking concept allows a tidy workspace and you can arrange all tool windows as you like. All changes you apply to the layout will be saved for the next start of the Toolbox. All tool bars are, like the Tool Windows, draggable and dockable. If they are not docked you can close them to get more free work space. To reactive a closed toolbar you can go the "Tool Bars" entry in the "View" menu and enable the toolbar. The Sentinel Toolbox commands are grouped in multiple tool bars by their purpose and include standard, layers, analysis, tools and windows toolbars.

SNAP Toolbars

In the first lesson we will use Landsat 8 data from USGS EarthExplorer website.

Earth Explorer: the main page

First of all you need to Create an Account or Login. At the top right corner of the website, click on Register to create a new account, or Login if you are an existing user. As a new user, once you click on Register, you will be required to type in your preferred username and password. Following these requirements, fill in your preferred username and password and click Continue.

After successfully registering, you will be required to fill in some information to better understand how the data you are downloading is being utilized. Read each of the questions and choose the appropriate answers for you. Submit your registration after filling in all the required details successfully. You will be required to confirm and activate your account by clicking the link in the email. Click on Return to Login Page when you have activated your account to sign in.

Now that you are logged in, you are ready to search for and download Landsat Imagery. In the Search Criteria Tab, you need to specify your Area of Interest (AOI), and you can do this in multiple ways:

  • Geocoder: This allows you to select a geocoding method. You can either use Feature (GNIS), Address/Place, or Path/Row to choose your AOI
  • KML/Shapefile Upload: You can upload a KML/shapefile of your AOI if you already have one
  • Map Interface: You can zoom in to your AOI on the map interface and draw a shape to select your AOI
  • Coordinates: You can also manually enter the coordinates of your area of interest if you have the coordinates

After defining your AOI, you can choose a date range, a cloud cover range, and your preferred result options to streamline your search.

It is now time to select the data you want to download. You can use the Data Sets Search tab to search for it or you scroll down and navigate to the data category. In this case, we are downloading Landsat 8 Imagery. You will observe that there are two levels (Level 1 and Level 2) in Collection 2. Level 2 is mostly used because it is atmospherically corrected (Surface Reflectance) data. Check Landsat 8 C2/L2 data.

Data Sets Searching

Now that we have selected the dataset and defined our date and cloud cover range, click the "Results" button. This will update your search results to return results that match your query. When results are returned, it is important to check for some details before you proceed to download. You want to make sure the results returned actually cover your area of interest. You can confirm this by checking the footprint of the imagery. In the image below, checking the footprint shows that this particular imagery result does not cover my AOI. The red color is my AOI while the green color is the footprint of the first imagery. Do this until you find one that actually covers it.

Satellite Images Interactive Searching

Once you find one that matches your preference, you also need to check the metadata to see the “data” about your data. Here, you will see the imagery date, the time of acquisition, the cloud cover, and other information.

Image Parameters

When everything looks good, you can then proceed to download.

1.4 Data Import

Back to the beginning ⇡

Once you have installed SNAP, you can import the data you downloaded. There are two main ways of loading data into SNAP. To import the product, you can go to File > Open Product then navigate to the product file and open it.

The second way is to go to the File > Import. Depending on your choice a window will be opened which pre defines the file type which is required to load a certain data format (for example .xml, .hdr, .safe, or .zip). Loading data via File > Import is recommended, because it grants that the correct file is selected during the import so that all metadata of the input are interpreted correctly by SNAP.

Import Data into SNAP

When the product has loaded, it will appear in the table of contents (TOC) on the left-hand side of the interface. You can expand the product using the “+” icon to the left of the product name. You will find the individual band data in the folder labeled ‘Bands’. Double click on the band name to view it in the workspace viewer window.

List of available images from source

1.5 Raster Cropping

Back to the beginning ⇡

You may need to subset your data, as is unlikely that you will require the whole Sentinel 2 scene. The Subset tool can be used to clip the image to your specific area of interest (AOI). It may be best to subset your image before you carry out any analysis tasks, as a smaller image will take less computing power to process. To subset your data, click the product name in the TOC to highlight it, then click Raster on the toolbar and select Subset from the dropdown menu.

Subset Tool Location in SNAP Interface

In the Product Subset window view, further adjust your subset by using your mouse to move the sides on the blue rectangle outline. You can fine-tune your subset by entering values in the four boxes under the Pixel Coordinates and Geo Coordinates tabs. Manually selecting geo-coordinates is helpful if you wish to create identical subsets of multiple products.

Subset Tool Menu

Or you can try another way - using your mouse-wheel to zoom and left-click to drag (to do this, you must select the panning tool), center your view on the area you wish to study. Right-click in the view area and select Spatial Subset from View.

Subset From View Tool Location

When you have defined the AOI, click on the "Band Subset" tab. Here, you can select the bands that you want to include in the subset image.

Bands choosing from Subsetting Image

After clicking OK, the subset product will be added to the Table of Contents with the prefix subset_. Double-click on this subset product to expand, then expand the Bands folder and double-click on the one of bands to view the image.

1.6 Data visualization

Back to the beginning ⇡

Now that the data has been imported and subsetted, you can carry out processing and analysis tasks on the data. Firstly, to create an RGB (Red, Green, Blue) image, also known as a True Color Image, right click on the product name in the TOC and select Open RGB Image Window from the drop-down menu.

RGB Viewer Button

A window will open, allowing you to select the spectral bands that you would like to assign to each color (red, green, blue). If you choose Sentinel 2 MSI Natural Colors as the profile, the bands will be automatically assigned, with red corresponding to band 4, green corresponding to band 3, and blue corresponding to band 2. Alternatively, you can manually assign the bands if desired. Once you are satisfied with the assignment, click "OK". Finally, the RGB image will be loaded into the viewing window.

RGB Channels chooser

To create a stacking image file you need to use a Collocation tool. The Collocation Tool can be invoked from the tool menu in the Sentinel Toolbox by selecting Processing > Geometric Operations > Collocation. In the Source Products group you need to specify the reference product. The combo box presents a list of all products opened in SNAP. You may select one of these or, by clicking on the button next to the combo box, choose a product from the file system.

Then in Target Product Group Name you should specify the name of the target product. “Save as” checkbox is used to specify whether the target product should be saved to the file system. The combo box presents a list of file formats. “Open in SNAP” checkbox is used to specify whether the target product should be opened in SNAP after processing. When the target product is not saved, it is opened in SNAP automatically.

Each product consists of different components, such as bands, flag codings and bitmask definitions. In order to avoid naming conflicts between reference and secondary product components, the user can specify a renaming pattern. By checking or unchecking option "Rename reference components" the automatic renaming of reference product components can be activated or deactivated, respectively. If activated, all components of the reference product are renamed according to the pattern given in the text field next to the check box. When all information is complete, click "Run".

File Formats for Export
